# Architecture

## Цель

Создать модульную структуру, разделяющую:
- обработку событий и маршрутизацию (`bot/core`)
- работу с базой данных (`bot/database`)
- интеграцию с VK API (`bot/vk_api`)
- формирование пользовательского интерфейса (`bot/ui`)
- бизнес-логику поиска и избранного (`bot/user`)

## Основные слои

### `main.py`
Минимальная точка входа, которая проверяет конфигурацию и запускает бот.

### `bot/core`
- `bot/core/bot.py` — стартует цикл прослушивания событий.
- `bot/core/handlers.py` — маршрутизирует входящие сообщения по состояниям.
- `bot/core/states.py` — содержит константы состояний.

### `bot/database`
- `bot/database/models.py` — SQLAlchemy ORM-модели: `Users`, `Status`, `Favorite`.
- `bot/database/session.py` — создаёт движок и сессию SQLAlchemy.
- `bot/database/init_db.py` — утилита для инициализации таблиц.
- `bot/database/repositories/` — отдельные репозитории для доступа к данным.

### `bot/vk_api`
- `bot/vk_api/client.py` — создаёт VK-сессию.
- `bot/vk_api/search.py` — поиск пользователей по критериям.
- `bot/vk_api/photos.py` — получение фотографий анкеты.
- `bot/vk_api/users.py` — получение информации о пользователе и работе с городами.

### `bot/ui`
- `bot/ui/keyboard.py` — билдер клавиатур.
- `bot/ui/messages.py` — шаблоны сообщений и подсказки.
- `bot/ui/formatter.py` — форматирование текста анкеты.

### `bot/user`
- `bot/user/preferences.py` — логика формирования критериев поиска.
- `bot/user/favorites.py` — логика избранного.
- `bot/user/service.py` — общий сервисный слой пользователя.

## Поток данных

1. Пользователь отправляет сообщение.
2. `bot/core/handlers.py` определяет состояние и вызывает бизнес-логику.
3. Данные сохраняются/читаются из PostgreSQL через `bot/database/repositories`.
4. Поиск и данные пользователей берутся через `bot/vk_api`.
5. Результат отправляется пользователю вместе с клавиатурой.

## Преимущества архитектуры

- Модульность и разделение ответственности.
- Легче тестировать отдельные слои.
- Большая часть логики отделена от прямого VK API и от SQLAlchemy.
