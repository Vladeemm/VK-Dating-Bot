1. АРХИТЕКТУРНЫЕ ПРОБЛЕМЫ (КРИТИЧЕСКИЕ)
=============================================================================

1.1. ОТСУТСТВИЕ МОДУЛЬНОЙ СТРУКТУРЫ
     ├─ Все файлы разбросаны в корне проекта
     ├─ Нет разделения на компоненты (core, database, vk_api, ui)
     ├─ Сложно масштабировать и поддерживать
     └─ Рекомендация: Реструктурировать согласно плану в plan.md

1.2. ДУБЛИРОВАНИЕ СИСТЕМЫ ПОДКЛЮЧЕНИЯ К БД
     ├─ create_tables.py — использует psycopg2 напрямую
     ├─ DatingBotBase.py — использует SQLAlchemy ORM
     ├─ Проблема: две разных системы для одной БД
     ├─ Следствия:
     │  - Конфликт при миграции схемы
     │  - Сложность в поддержке
     │  - Непредсказуемое поведение
     └─ Решение: использовать только SQLAlchemy и убрать дублирование

1.3. КОНСТАНТЫ СТАТУСОВ В ДВУХ МЕСТАХ
     ├─ statuses.py — определение констант
     ├─ main.py — дублирование констант
     ├─ Проблемы:
     │  - Несинхронность при изменениях
     │  - Конфликты при слияниях
     │  - Непредсказуемость
     └─ Решение: единственный источник в bot/core/states.py

1.4. СМЕШАННЫЕ ОТВЕТСТВЕННОСТИ В ОДНОМ ФАЙЛЕ
     ├─ actions.py содержит UI, БД и бизнес-логику
     ├─ Сложно переиспользовать код и поддерживать
     └─ Решение: разделить на bot/database, bot/vk_api, bot/user, bot/ui

=============================================================================
2. ЛОГИЧЕСКИЕ ОШИБКИ И НЕДОСТАТКИ (КРИТИЧЕСКИЕ)
=============================================================================

2.1. НЕПРАВИЛЬНАЯ СТРУКТУРА УСЛОВИЙ В main.py
     ├─ elif внутри if блока `if user_status.step == VIEWING_QUESTIONNAIRES`
     ├─ Следствие: кнопка "⏩ Далее" может не обрабатываться правильно
     └─ Решение: вынести логику в отдельные обработчики

2.2. НЕПОЛНАЯ РЕАЛИЗАЦИЯ preference_formation
     ├─ Нет завершения и перехода на VIEWING_QUESTIONNAIRES
     └─ Следствие: пользователи могут застрять на этапе выбора критериев

2.3. ОТСУТСТВИЕ РЕАЛИЗАЦИИ ИЗБРАННОГО
     ├─ Добавление в избранное не реализовано
     ├─ get_favorite_questionnaire отправляет заглушку
     ├─ Удаление из избранного отсутствует
     └─ Решение: реализовать CRUD для favorites и логику показа

2.4. ОТСУТСТВУЕТ ОБРАБОТКА ОШИБОК VK API
     ├─ Нет try-except блоков для VK API запросов
     ├─ Программа может упасть при сетевой ошибке
     └─ Решение: добавить обработку исключений и retry

=============================================================================
3. НЕДОСТАЮЩАЯ ФУНКЦИОНАЛЬНОСТЬ
=============================================================================

3.1. get_random_id() НЕ ОПРЕДЕЛЕН
     ├─ bd_actions.py импортирует get_random_id из actions.py
     ├─ actions.py такой функции не содержит
     └─ Следствие: ImportError / NameError при запуске

3.2. ОТСУТСТВУЕТ ПАГИНАЦИЯ ИЗБРАННОГО
     ├─ По спецификации нужно страницы в избранном
     ├─ В текущем коде нет управления страницами
     └─ Решение: добавить page_number и навигацию

3.3. ОТСУТСТВУЕТ УДАЛЕНИЕ ИЗ ИЗБРАННОГО
     ├─ Нет функции удаления
     ├─ Нет кнопки обработки
     └─ Решение: реализовать в favorite_repo

3.4. ОТСУТСТВУЕТ ВАЛИДАЦИЯ ВХОДНЫХ ДАННЫХ
     ├─ возраст проверяется на `isdigit` и диапазон
     ├─ нет защиты от отсутствующих полей
     └─ Решение: добавить явную валидацию и fallback
